# 進度暫存規則（cline_progress.md）

## 目的
為避免長對話造成上下文遺失（歷史截斷、Plan → Act 細節遺漏、Lost in the Middle），所有實作任務都必須維護一份外部進度記憶檔：`@/cline_progress.md`。

---

## 強制規範

1. **第一次開始實做（Act Mode）時**，必須先清空 `@/cline_progress.md` 內容，再寫入本次任務的最新進度框架。
2. `@/cline_progress.md` 必須至少包含：
   - 任務標題 / 任務描述（簡短）
   - 待完成步驟（可勾選清單）
   - 進行中步驟
   - 已完成步驟（含完成摘要）
   - 變更紀錄（每次更新一行：時間 + 動作）
   - 風險 / 阻塞（若有）
3. 每完成一個步驟與詳細步驟，**必須立即更新** `@/cline_progress.md`：
   - 將對應待辦由 `- [ ]` 改為 `- [x]`
   - 補上完成摘要
   - 若計劃變更，立即增修刪改步驟清單
4. 任務範圍變更時，必須同步重排待辦順序，並在「變更紀錄」說明原因。
5. 全部完成後，需再次逐條檢查待辦與完成摘要，確認無遺漏後，在檔案最後一行寫入固定標記：
6. 修復任務也要更新進度暫存

`=== 全部完成 ===`

---

## 建議模板（可直接貼到 `@/cline_progress.md`）

```md
# 任務進度暫存

## 任務
- 名稱：{task_name}
- 目標：{task_goal}
- 開始時間：{yyyy-mm-dd hh:mm}

## 待辦清單(重要細節不可缺失，可能遇見的問題要附上，步驟避免過於簡略)
- [ ] 步驟 1：...
  - [ ] 詳細步驟 1-1：...
    - 細節1-1-1
    - 細節1-1-2
    ...
  - [ ] 詳細步驟 1-2：...
    - 細節1-2-1
    - 細節1-2-2
    ...
  ...
- [ ] 步驟 2：...
  - [ ] 詳細步驟 2-1：...
    - 細節2-1-1
    - 細節2-1-2
    ...
  - [ ] 詳細步驟 2-2：...
    - 細節2-2-1
    - 細節2-2-2
    ...
  ...
- [ ] 步驟 3：...
  - [ ] 詳細步驟 3-1：...
    - 細節3-1-1
    - 細節3-1-2
    ...
  - [ ] 詳細步驟 3-2：...
    - 細節3-2-1
    - 細節3-2-2
    ...
  ...
...

## 進行中
- 目前處理：...

## 已完成
- （完成後填寫）

## 變更紀錄
- {time} 初始化檔案並建立待辦

## 風險與阻塞
- 無 / {描述}

## 用戶需要進行的檢查與確認
- {描述}
...
```

> 注意：`=== 全部完成 ===` 只能在所有步驟皆完成且逐條核對無誤後追加在檔案末尾。
> 需要在完成後有修復或新增功能，需要移除末尾的`=== 全部完成 ===`，並且新增未完成的步驟。

---

## 執行節奏（操作準則）

- **初始化**：清空檔案 → 寫入任務與待辦。
- **過程中**：每完成一段可驗證步驟就更新一次`@/cline_progress.md`並確認。
- **上下文重整**：確認當前進度。
- **有變更**：即時增修刪改步驟清單與紀錄。
- **最終收尾**：逐條核對 → 追加 `=== 全部完成 ===`。
